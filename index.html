Workflow Github

un petit récap du workflow github pour projet en équipe. N'hésitez pas à me dire si il faut modifier, si il manque des trucs etc ...
(je suis pas du tout experte en la matière)
* remote = par ex git@github.com:Louya64/WorkflowGithub.git
* En vert, ce qu'on tape dans le terminal

  1. sur github, on crée le repo pour avoir une adresse remote qui ressemble à ça: git@github.com:Louya64/WorkflowGithub.git ou ça: https://github.com/Louya64/WorkflowGithub.git (vérif que c'est ok pour les 2 avec Lydie ou Joseph)
  2. sur github on crée une branch "dev" (ou autre nom ? voir Lydie ou Joseph) en ouvrant l'onglet main voir la photo
  3. sur github, on invite ses potes en collaborateurs (sur le repo concerné settings manage access)

  4. si on n'a pas encore de dossier: en local, on se place dans le dossier parent et on clone le repo : git clone remote puis on se déplace dans le dossier : cd nomDuDossier puis : git init

    si on a déjà un dossier: en local, on va dans son dossier : cd cheminDuDossier puis : git init puis git remote add origin remote
    (le mot "origin" est une convention, on pourrait remplacer par "patate" - c'est un peu la variable qui contient l'adresse github du repo)

  5. on se déplace sur la branch dev : git checkout dev on peut vérifier le nom de la branch sur laquelle on est : git branch puis, pour être sûr d'être à jour par rapport à dev : git pull remote (pas sure que ce soit remote on peut pull nomBranch? on peut "git pull" tout court ?)
  6. on crée sa branch genre "header" ou "fixBugResponsive" etc selon ce qu'on va faire : git branch fixBugResponsive puis on se déplace dessus : git checkout fixBugResponsive
  7. on bosse dur sur son vscode
  8. une fois fini le morceau sur lequel on travaille : git add . ou git add leFichierNana leFichierTruc (faut des espaces entre les fichiers?)
  9. puis git commit -m "message genre fix bug responsive in pageContact..."
  10. ! Attention pushez sur VOTRE branch (pas sur dev ni main) : git push origin fixBugResponsive
  11. sur github, on fait une pull request de nomDeVOTREbranch vers dev voir la photo
  12. on résout les conflits si y'en a puis on merge sur dev (désolée c'est flou ici pour moi donc pas très détaillé - bonne chance pour les conflits !!)
